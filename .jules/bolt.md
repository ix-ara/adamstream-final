## 2025-05-14 - Deferred Row Rendering for DOM and Network Efficiency
**Learning:** In media-heavy applications like AdamStream, rendering hundreds of movie cards (even with lazy-loaded images) significantly inflates the DOM size and impacts Time to Interactive (TTI). Implementing deferred rendering at the *row* level using `IntersectionObserver` with a `rootMargin` (e.g., 300px) effectively limits the active DOM node count and staggered image requests without sacrificing user experience. A `min-height` on the row container is essential to prevent Cumulative Layout Shift (CLS) when the cards are eventually injected.
**Action:** Prioritize deferred component rendering over simple image lazy-loading for off-screen content in large lists or grids. Always pair with CSS-reserved space to maintain layout stability.
